# Ticket Please – Laravel API Master Class Practice

**Ticket Please** is a ticketing system API built as part of the Laracasts _Laravel API Master Class_ series.  
This repository documents my progress while designing, versioning, building, securing, and documenting a production-ready RESTful API using Laravel.

Series link: https://laracasts.com/series/laravel-api-master-class

## Project Overview

Ticket Please is a backend API for managing tickets and users, designed with proper API structure and best practices in mind. The project is built incrementally, following real-world API design principles.

Core concepts implemented in this project:

- API design and logical URL structure
- API versioning strategy
- Consistent JSON response formatting
- Token-based authentication with Laravel Sanctum
- Token revocation and expiration handling
- Resource transformation using JsonResource
- Conditional attribute inclusion
- Query filtering and sorting
- Nested resources
- CRUD operations (POST, PUT, PATCH, DELETE)
- Authorization using Policies
- Token abilities and granular permissions
- Principle of Least Privilege
- Global exception handling and standardized error responses
- Automated API documentation using Scribe

## Tech Stack

- Laravel 12
- Laravel Sanctum (API authentication)
- Postman (API testing)
- Scribe (API documentation)
